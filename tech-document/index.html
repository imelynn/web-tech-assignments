File Structure

File structure is important and critical to an html document functioning properly within the browser. If the filing of the html document itself, in addition to stylesheets, images, and javascripts are not correct, the browser will not read it correctly. Elements can appear all over the place, out of context, unordered, or missing entirely from the user’s viewport once rendered by the user agent (browser).

The user agent must know the file path in order to locate and access files associated with the html document. A file path describes the location of a file in a web site's folder structure.

File paths are used when linking to external files like:

  • Web pages
  • Images
  • Stylesheet
  • JavaScripts
Best practice is to keep all files associated with the webpage including the html file, local document linkages, images, and stylesheets in the same folder.

Images can be tricky to access depending on how they are linked form the source document. Outside of URL linked images, its easier and best practice to just keep all images in the webpage root folder. Decide whether images will have their own folder of reside in the same folder as the html document in the root folder.

Code Template

Every program must include !DOCTYPE tag, it is NOT CASE SENSITIVE.

!DOCTYPE is not a tag it is a set of instructions called declarations that are provided to the browser. Having a declaration document attached ensures that it appears the same in different browsers. The declaration document is called document type definition or DTD.

Legacy HTML Versions (before HTML Version 5) required a URI link to the actual DTD. Declaring it in the html document looks as such:

!DOCTYPE html PUBLIC "-//W3C//DTD HTML 4.01 Transitional//EN" "http://www.w3.org/TR/html4/loose.dtd"

  • The DTD is a compilation of all the entity sets used in that version of HTML.
  • Entity sets are a collections of entities that share the same attributes.

Example:

In HTML, a character is an entity type: less than (), greater than (>), and ampersand (&), are all entities that share the same character attributes. Together they represent an entity set. Every entity has an entity type, name, and number. Refer to list below:

HTML Version 4.01 has three DTDs and three entity set collections and an URI should be provided in the !DOCTYPE tag to access them:

  • "strict.dtd" -- default strict DTD
  • "loose.dtd" -- loose DTD
  • "frameset.dtd" -- DTD for frameset documents
  • "HTMLlat1.ent" -- Latin-1 entities
  • "HTMLsymbol.ent" -- Symbol entities
  • "HTMLspecial.ent" -- Special entities

In newer versions of HTML, like html 5 the !DOCTYPE declaration does not require an URI only: DOCTYPE html

Must Have Tags, Well Somewhat

Precursor to the information in this section:

'html' and 'body' tags are not required in HTML 5, the document will still run fine in browsers like Chrome and Firefox but may break in others. **However, not using the tags comes with conditions that must be implemented in order to get the program to run efficiently**.

One such condition:

e.g. The html start tag is optional if the first thing inside the html element is a comment, otherwise browser may not render document properly.

So the choices are:

  • Not use the tags but learn the necessary conditions because you’re just that bored with lots of time on your hand.
  • Stick to what you know, use the tags, and live a simple life. There is beauty and profound peace in simplicity.
Some of the primary tags for an html document are:
  • html
  • head
  • title
  • link
  • style
  • meta
What to know about the primary tags:

HTML

  • After the document type declarations (DTDs), the rest of the program is contained in the HTML tag.
  • Start and End tags are optional.

HEAD

  • Start and End tags are optional.
  • The head element contains information about the current document, such as its title, keywords that may be useful to search engines, and other data that is not considered website content like links, metadata, and style sheets.
  • Elements that appear in the head are not generally compiled as website content. The browser may, however, make information in the head available to users through other mechanisms.

TITLE

  • Start tag: required, End tag: required
  • Every HTML document must have a title element in the section, all other elements are optional.
  • Authors should use the TITLE element to identify the contents of a document.
  • Since users often consult documents out of context, authors should provide context-rich titles. Thus, instead of a title such as "Welcome", which doesn't provide much contextual background, authors should supply a title such as "The Healing Heart Sanctuary," instead.
  • There are two uses of title, one is global (element, only appears once in the header) and the other is local (attribute, appears in any number of elements, mostly as a descriptor).

Example:

A href="Intuitive Lifestyle Brand | Home" title="Businesses with an intuitive foundation for doing business.">

Some other optional elements included in the head tag:

LINK

  • Start tag: required, End tag: optional
  • Link element indicates a relationship between the document and some other object, such as link to style sheet.

Example:

link rel="stylesheet" href="styles.css"

STYLE

  • Start tag: required, End tag: required.
  • Provides styling elements directly in the code.

Example:

body {background-color: powderblue;}
h1 {color: blue;}
p {color: red;}

META

  • Start tag: optional, End tag: optional
  • Metadata is data (information) about the document such as character set, page description, keywords, and author of the document. and viewport settings.
  • Metadata will not be displayed on the page, but will be parsed by agents such as the browser (how to display content or reload page), search engines (keywords), and other web services.
  • Viewport is a very important function, designers take control over the viewport (the user's visible area of a web page), through the meta tag.
  • Hypertext Transfer Protocol (HTTP) is also managed in the meta tag.

Examples:

meta charset="UTF-8"
meta name="description" content="Getting Started with HTML"
meta name="keywords" content="HTML, CSS, JavaScript, DTD, Technical Document"
meta name="author" content="Gena Hurst"
meta name="viewport" content="width=device-width, initial-scale=1.0"

There are many uses for the meta tag that goes beyond the scope of this document, please refer to the sources below for additional information.

BODY

  • Start tag: optional, End tag: optional
  • 'frameset' is the legacy version of 'body'.
  • The body of a document contains the website content. The content may be presented by a user agent (browser) in a variety of ways:
    1. For visual browsers, you can think of the body as a canvas where the content appears: text, images, colors, graphics, etc.
    2. For audio user agents, the same content may be spoken.
  • Since style sheets are now the preferred way to specify a document's presentation, the representational attributes of the body have been deprecated (archived).
  • All visual content that will appear on the webpage is contained in the body tags.

Hierarchy

An HTML document is like a big family tree, with parents, siblings, children, ancestors, and descendants. It comes from the ability to nest HTML elements within one another.

  • Ancestor – Represented by the HTML tag. An ancestor refers to any element that is connected but further up the document tree - no matter how many levels higher.
  • Descendant – Represented by the HEAD/BODY tag as a descendant of HTML.
    A descendant refers to any element that is connected but lower down the document tree - no matter how many levels lower.
  • Parent/Child – Represented by HEAD (Parent) and TITLE (Child) tags.
    A parent is an element that is directly above and connected to an element in the document tree.
    A child is an element that is directly below and connected to an element in the document tree.
  • Siblings – Represented by the TITLE, LINK, STYLE, and META tags.
    A sibling is an element that shares the same parent with another element.
Closing

HTML documents all close with the ending HTML tag."
